What is a CNN?

A Convolutional Neural Network (CNN) is a type of deep learning algorithm that is particularly powerful for analyzing visual imagery.

CNNs are designed to automatically and adaptively learn spatial hierarchies of features from input images through backpropagation.

Note: CNNs are widely used in image recognition, video analysis, medical image analysis, and natural language processing.

Why Use CNNs?

Traditional neural networks don't scale well to full images. CNNs solve this problem by:

  • Reducing the number of parameters through weight sharing
  • Automatically detecting important features without human supervision
  • Being translation invariant (detecting objects regardless of position)
  • Handling images of varying sizes

CNN Architecture

A typical CNN architecture consists of the following layers:

Layer Description Purpose
Input Layer Receives the raw pixel values of the image Data entry point
Convolutional Layer Applies filters to detect features like edges Feature extraction
Pooling Layer Reduces spatial dimensions of feature maps Dimensionality reduction
Fully Connected Traditional neural network layers Classification
Output Layer Produces final predictions Results

The Convolution Operation

The convolution operation involves sliding a filter (kernel) over the input image and computing the dot product at each position.

Example

A simple 3x3 edge detection kernel:

# Edge Detection Kernel
kernel = [
    [-1, -1, -1],
    [-1,  8, -1],
    [-1, -1, -1]
]
# This kernel highlights edges in images
# by detecting rapid changes in pixel intensity

Pooling Layers

Pooling layers reduce the spatial dimensions of the feature maps, which helps to:

  • Reduce computational cost
  • Reduce overfitting
  • Make the detection more robust to position changes

Types of Pooling

Max Pooling: Takes the maximum value from each region

Average Pooling: Takes the average value from each region

Global Pooling: Reduces each feature map to a single value

Example

Max Pooling with a 2x2 window:

# Max Pooling Example
import torch.nn as nn
# Define max pooling layer
max_pool = nn.MaxPool2d(kernel_size=2, stride=2)
# Input: 4x4 feature map
# Output: 2x2 feature map
# Example:
# Input:          Output:
# [1, 2, 3, 4]    [6, 8]
# [5, 6, 7, 8] -> [14, 16]
# [9, 10, 11, 12]
# [13, 14, 15, 16]

Complete CNN Example

Here's a complete CNN implementation using PyTorch for image classification:

Example

import torch
import torch.nn as nn
import torch.nn.functional as F
class CNN(nn.Module):
    def __init__(self):
        super(CNN, self).__init__()
        # Convolutional layers
        self.conv1 = nn.Conv2d(1, 32, kernel_size=3, padding=1)
        self.conv2 = nn.Conv2d(32, 64, kernel_size=3, padding=1)
        
        # Pooling layer
        self.pool = nn.MaxPool2d(2, 2)
        
        # Fully connected layers
        self.fc1 = nn.Linear(64 * 7 * 7, 128)
        self.fc2 = nn.Linear(128, 10)
        
        # Dropout for regularization
        self.dropout = nn.Dropout(0.25)
    
    def forward(self, x):
        # First conv block
        x = self.pool(F.relu(self.conv1(x)))
        
        # Second conv block
        x = self.pool(F.relu(self.conv2(x)))
        
        # Flatten
        x = x.view(-1, 64 * 7 * 7)
        
        # Fully connected layers
        x = F.relu(self.fc1(x))
        x = self.dropout(x)
        x = self.fc2(x)
        
        return x
# Create model instance
model = CNN()
print(model)

Activation Functions in CNNs

Activation functions introduce non-linearity into the network:

Function Formula Use Case
ReLU max(0, x) Most common, hidden layers
Leaky ReLU max(0.01x, x) Prevents dying neurons
Sigmoid 1 / (1 + e^-x) Binary classification
Softmax e^xi / Œ£e^xj Multi-class output

Warning: ReLU can cause "dying neurons" where neurons output 0 for all inputs. Consider using Leaky ReLU or other variants.

Training a CNN

Training a CNN involves the following steps:

  1. Forward Pass: Input data flows through the network
  2. Loss Calculation: Compare predictions with actual labels
  3. Backward Pass: Calculate gradients using backpropagation
  4. Weight Update: Adjust weights using an optimizer (e.g., Adam, SGD)
  5. Repeat: Continue for multiple epochs until convergence

Example

Training loop in PyTorch:

# Training loop
criterion = nn.CrossEntropyLoss()
optimizer = torch.optim.Adam(model.parameters(), lr=0.001)
for epoch in range(num_epochs):
    for images, labels in train_loader:
        # Forward pass
        outputs = model(images)
        loss = criterion(outputs, labels)
        
        # Backward pass
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()
    
    print(f'Epoch [{epoch+1}/{num_epochs}], Loss: {loss.item():.4f}')
